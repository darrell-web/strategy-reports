FND-01Verify age-gate Googlebot allowlist (cannabis-binary)P0★★★★★ if blocking

If the age-gate is implemented client-side without a Googlebot allowlist, the homepage Google crawls is the empty age-verification interstitial — not the new H1 from DEV-01. Every Pillar 1 deploy is wasted if this is broken. Single biggest rank mover when broken.

OwnerSEO/Analytics + Dev
DeadlineEOD May 5
Time2 hrs
Expected lift if broken+10 to +20 SoLV

Steps

  1. In GSC, run URL Inspection on https://joyleaf.com/. Click "Test live URL" → after the test runs, click "View tested page" → switch to the HTML tab.
  2. Search for the new H1 ("Roselle's Trusted Cannabis Dispensary") and the LocalBusiness JSON-LD <script> block. Both must be present in the rendered HTML Google sees.
  3. If either is missing, the age-gate is blocking Googlebot. Pull joyleaf.com server access logs for the last 7 days. Filter for User-Agent matching Googlebot.
  4. For each Googlebot request to /, confirm the response served the actual page HTML (200 status, ~30-100KB body). If the response is the lightweight age-gate page (small body, redirect, or interstitial JS-only response), the allowlist is missing.
  5. If broken: implement a server-side Googlebot allowlist. The pattern is: detect Googlebot|Bingbot|DuckDuckBot in User-Agent; if matched, bypass the age-gate cookie/session check and serve the full page.
  6. Deploy. Re-run URL Inspection. Confirm the H1 + schema are visible in the rendered HTML tab.
  7. Submit homepage + /dispensary-location URLs in GSC for re-indexing.
Why this matters Cannabis Creative + Whitespark consensus: the age-gate is the most common stealth indexation killer in cannabis. Joyleaf may already have this configured correctly — if so, FND-01 closes in Step 2 (verify pass). If not, fixing it is the highest-leverage single change in May.
Acceptance criteria
  • GSC URL Inspection live test of joyleaf.com/ shows the new H1 and LocalBusiness JSON-LD in the rendered HTML.
  • Server logs for the last 7 days show Googlebot receives 200 responses with full page bodies (not the age-gate interstitial).
  • If a fix was deployed, re-run after deploy and confirm the rendered HTML now includes the H1 + schema.
  • Homepage + location URLs submitted for re-indexing.
FND-02Audit Dutchie iframe vs. Plus / native menu (cannabis-binary)P0★★★★★ if iframe

If joyleaf.com/menu (or /shop) is iframe-loaded via the default Dutchie embed, every product page is invisible to Google. Cannabis Creative + CannaPlanners consensus: highest-leverage cannabis-specific SEO fix available.

OwnerSEO/Analytics + Dev
DeadlineAudit EOD May 8 · Migration plan EOD May 31
Time4 hrs audit + variable migration
Expected lift if iframe+10 to +20 SoLV (post-migration)

Steps

  1. Open joyleaf.com/menu (or whichever URL serves the Dutchie menu). View page source (right-click → View Page Source — NOT Inspect).
  2. Search source for <iframe. If found, look at the src — if it points to dutchie.com or a Dutchie subdomain, the menu is iframe-loaded and the products are invisible to Google.
  3. Run the URL through Lighthouse (Chrome DevTools → Lighthouse → SEO audit). The "Page has no robots tag" + "Document has a meta description" + "Links have descriptive text" tests will surface menu-content issues if the iframe is the only source of product info.
  4. Run Schema validator on the parent /menu page. If no Product schema is detected, the products are invisible to Google's structured-data layer.
  5. Document the finding in FND-02-dutchie-audit.md in the Joyleaf project folder. Include: iframe present (Y/N), iframe src, parent-page Product-schema present (Y/N), Lighthouse SEO score, recommended remediation path.
  6. If iframe-loaded: prepare a migration recommendation — Option A: enable Dutchie Plus (Dutchie's SEO-friendly menu product, ~1-line config + paid tier); Option B: build a native menu sourced from Dutchie's API. Quote both; loop in Account Lead.
  7. If already migrated to Dutchie Plus or native: confirm Product schema is present on parent page; close FND-02 as already-good.
Why this matters Joyleaf's strain catalog represents a substantial long-tail KW universe ("[strain name] near me", "[strain name] roselle nj"). If the menu is iframe-loaded, none of that traffic is reachable. The Dutchie Plus migration is the single highest-leverage cannabis-vertical fix; it pays for itself in 90 days for any dispensary with material strain-level search volume.
Acceptance criteria
